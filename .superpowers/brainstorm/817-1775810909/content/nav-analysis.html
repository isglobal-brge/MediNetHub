Sidebar vs Top Bar: Analisis por pagina

Cual conviene mas en cada pagina de MediNetHub segun su contenido real

Pagina Contenido principal Necesidad de ancho Ganador Por que
Model Designer Canvas drag-and-drop + panel de capas + config lateral
CRITICA
 B Sidebar Se colapsa a 60px → canvas gana ~160px. Es la pagina que mas espacio necesita.
Dashboard Home 4 stat cards + activity feed + notifications
MEDIA
 B Sidebar Grid de 4 cards cabe bien. Sidebar da contexto de "donde estoy" siempre visible.
Training (tabla) Tabla de jobs con 5 columnas + botones de accion
ALTA
 = Ambas Sidebar colapsada (60px) pierde poco ancho. Tabla de 5 cols cabe en 1200px-60px = 1140px.
Training Dashboard Metricas en tiempo real + clientes + logs + grafico
ALTA
 B Sidebar Info densa pero vertical. Sidebar colapsada no molesta. Logs y metricas se benefician de scroll vertical.
Model Studio Sidebar de filtros + grid de model cards + tabs
MEDIA
 ⚠ Atencion Ya tiene sidebar propia de filtros (col-md-3). Con sidebar nav → doble sidebar. Solucion: integrar filtros en la nav sidebar o moverlos a dropdown/tabs.
Datasets Stepper + project selector + card list de datasets
BAJA
 B Sidebar Cards apiladas no necesitan mucho ancho. El stepper de workflow podria integrarse visualmente en la sidebar.
Experiment Detail Ranking de jobs + graficos de comparacion + metricas
ALTA
 = Ambas Graficos se adaptan al container. 60px menos no afecta significativamente.
Home / Login / Register Landing publica, formularios centrados
N/A
 Sin sidebar Pre-autenticacion. Sidebar no aparece. Solo top bar minima o layout limpio.
Profile / Notifications Formularios y listas simples
BAJA
 B Sidebar No necesitan ancho. Sidebar da navegacion contextual util.

Recomendacion: Sidebar Colapsable (B)
  • Gana en la pagina mas critica (Model Designer = maximo canvas)
  • Solo 1 conflicto solucionable: Model Studio → mover filtros a toolbar/dropdown
  • Paginas pre-auth (Home, Login, Register) no muestran sidebar — layout independiente
  • Escalable si anadis mas secciones en v0.3+

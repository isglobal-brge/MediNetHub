## Major Updates

### 1. Machine Learning Model Support (Major Feature)
**Status**: In Development


#### Description
Extension of the model designer to support traditional machine learning algorithms alongside existing deep learning capabilities.

#### Planned ML Algorithms
- **Support Vector Machines (SVM)**
  - SVM
- **Decision Trees**
  - CART (Classification and Regression Trees)
  - Configurable max depth, min samples split
- **Future Considerations**: Random Forest, Gradient Boosting

#### Technical Implementation
- New model type: `ml` (alongside existing `dl` type in ModelConfig)
- ML model builder module (`webapp/server_fn/ml_model_builder.py`)
- Federated aggregation strategies adapted for ML models
- UI updates in model designer for ML-specific parameters

#### Impact Areas
- `ModelConfig` model: Already supports `model_type` field with `ml` option
- `model_builder.py`: New ML model builder class
- `strategies.py`: ML-aware aggregation logic
- Model designer UI: New ML layer palette and parameter forms
- Training configuration: ML-specific hyperparameters

### 3. Enhanced Model Download System (Minor Feature)
**Status**: Planned


#### Description
Improved system for downloading trained models with better organization, metadata, and format options.

#### Planned Improvements
- **Multiple Download Formats**:
  - PyTorch state dict (.pth)
  - ONNX format (.onnx) for framework-agnostic deployment
  - TorchScript (.pt) for production inference
  - Architecture JSON with trained weights

- **Checkpoint Management**:
  - Already has `save_frequency` field in TrainingJob model
  - Implement periodic checkpoint saving during training
  - Download specific round checkpoints
  - Compare checkpoints across rounds

- **ModelArtifact System** (Already in models.py):
  - Leverage existing `ModelArtifact` model
  - Organize artifacts by type: `architecture`, `checkpoint`, `final_model`
  - Track file sizes and round numbers
  - User-specific storage paths

- **Download UI Enhancements**:
  - Model comparison before download
  - Metadata export (training config, metrics, timestamps)
  - Batch download for multiple checkpoints
  - Download history tracking

#### Technical Implementation
- Update `TrainingJob` save logic to create `ModelArtifact` entries
- ONNX export utilities for PyTorch models
- ZIP archive creation for bundled downloads
- Storage quota tracking via `UserProfile.storage_used`
